# Constraints & Conventions

## PHP & Language Rules

- **`declare(strict_types=1)`** is required on every PHP file — no exceptions.
- **PHP >= 8.4** is the minimum supported version. Do not use deprecated features or add workarounds for older PHP versions.
- All classes live in the `Mistralys\SeriesManager` namespace (or a sub-namespace). No global classes.
- **No MVC framework.** Do not introduce routing libraries, DI containers, or ORM layers. The architecture is intentionally simple page-based PHP.

---

## Security

- **This tool is not intended for public servers.** It has not been hardened for internet-facing use. Do not add features that assume a hostile network.
- The only authentication mechanism is a session-based password stored as a salted SHA-1 hash. `APP_PASSWORD` and `APP_SALT` live exclusively in `config-local.php`, which is gitignored.
- `config-local.php` must never be committed. Only `config-local.dist.php` (the template) belongs in version control.

---

## Configuration

- All environment-specific constants are defined in `config-local.php` under the `Mistralys\SeriesManager` namespace:
  - `APP_PASSWORD` — plain-text password for login
  - `APP_SALT` — arbitrary salt string for hashing
  - `APP_API_KEY` — TheTVDB API key
  - `APP_SUBSCRIBER_PIN` — TheTVDB subscriber PIN
  - `APP_LIBRARY_PATHS` — array of absolute paths to video library folders
- The constant `APP_ROOT` is defined in `index.php` as `__DIR__` and represents the project root. Use it for all absolute file paths within the codebase.

---

## Data Persistence

- **No database.** All data is stored in JSON files on disk.
- `data/series.json` is the master series list. It is written by `SeriesCollection::save()` and must never be edited by hand during a running session.
- `cache/library.json` is generated by `Library::createIndex()`. It must not be committed to git.
- `cache/{imdbID}-info.json` files are per-series TheTVDB caches. They are also not committed.
- The `data/` and `cache/` directories (except their `_readme.md` files) are gitignored.

---

## Autoloading

- Production classes go in `assets/classes/` and are autoloaded via Composer classmap.
- Test classes go in `tests/classes/` and are autoloaded via Composer classmap (dev).
- There is **no PSR-4** autoloading. Directory structure does not need to mirror namespace structure, but by convention it does (e.g., `Series\Series` → `assets/classes/Series/Series.php`).
- After adding a new class file, run `composer dump-autoload` to regenerate the classmap.

---

## Page Routing

- Pages are plain PHP files in `pages/`. The filename (without `.php`) is the page identifier used in the `?page=` query parameter.
- The valid page list is declared in `Manager::getPages()`. Pages not listed there (except the action pages below) will not be reachable.
- Action pages (`delete`, `edit`, `archive`, `unarchive`, `favorite`, `unfavorite`) require a valid `?id=` parameter containing an existing IMDB ID, validated by `Manager::getSelectedID()`.
- Sub-partials are placed in `pages/{page}/` and included explicitly from the parent page file (e.g., `library.php` includes `pages/library/{tab}.php`).

---

## Localization

- All user-facing strings must be wrapped in a localization helper:
  - `t('String')` — returns translated string
  - `pt('String')` — prints translated string
  - `pts('String')` — prints translated string followed by a space
  - `ptex('String', ...)` — prints translated string with printf-style substitutions
- Translation files live in `localization/`. Supported locales: `en_UK` (default), `de_DE`, `fr_FR`.
- The `localization/` folder is excluded from file finders (e.g., `Library::createIndex()`) via `excludeFolder('localization')`.
- The active locale is stored in `$_SESSION['locale']` and can be switched via the `selectLocale` GET/POST parameter.

---

## Library Indexing

- The library index is **built manually** by the user (resource-heavy scan). It is not triggered automatically on page load.
- Supported video file extensions: `mov`, `mpg`, `mpeg`, `mp4`, `avi`, `qt`, `mkv`.
- Folders named `.sync` and `.svn` are excluded during indexing.
- Series name detection from filenames works by:
  1. Normalizing the filename (lowercase, punctuation → spaces).
  2. Finding the episode marker pattern `S##E##` (case-insensitive, spaces tolerated).
  3. Everything before the marker becomes the series name.
- When a filename yields an unrecognized name, the user can add a **name alias** in `data/name-aliases.json` via the Library UI.

---

## TheTVDB Integration

- A TheTVDB subscription (API key + subscriber PIN) is required for data fetching. Without it, fetching is silently skipped.
- The API client (`adrenth/thetvdb2`) is created lazily in `Manager::createClient()` and cached for the request lifetime.
- The TheTVDB package is pulled from a custom VCS fork (`https://github.com/Mistralys/thetvdb2.git`); the VCS repository is declared in `composer.json`.
- Fetched data is cached per series in `cache/{imdbID}-info.json`. The cache is only refreshed when the user explicitly requests it (clear cache option on the fetch page, or per-series on the edit page).
- Episode names that are `tba`, `tbd`, `na`, or `n/a` (case-insensitive) are normalized to an empty string by `Series::filterName()`.

---

## Frontend

- The UI uses **Bootstrap 3** exclusively. Do not introduce Bootstrap 4/5 or any other CSS framework.
- Dark mode is toggled via `?toggleDarkMode=1` and stored in `$_SESSION['darkmode']`. When active, `bootswatch.min.css` replaces `bootstrap.min.css`.
- Icons are Bootstrap 3 Glyphicons (e.g., `glyphicon-star`, `glyphicon-ok-circle`).
- jQuery 1.11.2 is the only JavaScript library. Keep custom JS minimal; it lives in `js/add.js` and `js/list.js`.
- The `{SEARCH}` placeholder in `search-links.json` URL templates is replaced with `rawurlencode($searchTerm)`.

---

## Testing

- Tests extend `Mistralys\SeriesManagerTests\Classes\SeriesManagerSuite`, which provides `assertLibraryContainsFile()`.
- Test fixture files are stored in `tests/files/library-1/` and `tests/files/library-2/`.
- Run tests with: `composer exec phpunit` or `./vendor/bin/phpunit`.
- PHPStan static analysis is available: `./vendor/bin/phpstan analyse`.

---

## Version Management

- The version number is maintained in `changelog.md`. At boot, `Manager::__construct()` compares the file modification times of `changelog.md` and `version.txt`; if `changelog.md` is newer, it parses the latest version number and writes it to `version.txt` automatically.
- Do not manually edit `version.txt` — it is auto-generated.
- Current version: **3.0.1**
