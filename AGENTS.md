## 2. Manifest Maintenance Rules

When code changes, update manifest docs in the same task.

| Change Made | Documents to Update |
|---|---|
| Add/rename/remove class in `assets/classes/` | `api-surface.md` |
| Change public method/constant signatures | `api-surface.md` |
| Add/remove page in `pages/` or change routing behavior | `data-flows.md`, `constraints.md` |
| Change request/session parameters (e.g., `page`, `id`, `selectLocale`, dark mode toggles) | `data-flows.md`, `constraints.md`, `api-surface.md` (if API changed) |
| Modify auth logic, hashing, or login/logout flow | `data-flows.md`, `constraints.md`, `tech-stack.md` |
| Change JSON persistence format (`data/series.json`, aliases, cache schema) | `data-flows.md`, `constraints.md`, `api-surface.md` |
| Modify library indexing/parsing behavior or supported media extensions | `data-flows.md`, `constraints.md`, `api-surface.md` |
| Add/remove external dependency in `composer.json` | `tech-stack.md` |
| Change autoloading strategy or class locations | `tech-stack.md`, `constraints.md` |
| Add/modify tests or test layout | `tech-stack.md` |
| Update build/static analysis/test commands | `tech-stack.md`, `constraints.md` |
| Add/rename/remove manifest files | `docs/agents/project-manifest/README.md` and this `AGENTS.md` |

## 3. Efficiency Rules - Search Smart

- Finding where something lives: check `.context/project-folder-structure.md` first (auto-generated by CTX Generator).
- Understanding callable methods or class contracts: check `docs/agents/project-manifest/api-surface.md` first.
- Understanding behavior across files: check `docs/agents/project-manifest/data-flows.md` first.
- Checking allowed tools, frameworks, or version limits: check `docs/agents/project-manifest/tech-stack.md` and `docs/agents/project-manifest/constraints.md` first.
- Only read implementation files after manifest review does not answer the question.
- Do not infer architecture from one file when the manifest already defines it.

## 4. Failure Protocol & Decision Matrix

| Scenario | Action | Priority |
|---|---|---|
| Ambiguous requirement | Apply the most restrictive interpretation that preserves documented constraints; ask for clarification only when required to avoid harmful assumptions. | MUST |
| Manifest/code conflict | Trust the manifest, flag the code location as drift, and propose a code or manifest reconciliation step. | MUST |
| Missing manifest detail | Flag the documentation gap explicitly; do not invent undocumented behavior. | MUST |
| Missing `config-local.php` or required APP constants | Treat as environment/config issue; do not hardcode secrets or bypass config checks. | MUST |
| TheTVDB credentials unavailable (`APP_API_KEY`, `APP_SUBSCRIBER_PIN`) | Preserve graceful degradation; do not force API calls or fabricate remote data. | MUST |
| `cache/library.json` missing/stale | Direct user/flow to rebuild index; do not assume episodes are unavailable before indexing. | SHOULD |
| New class not autoloading | Check classmap conventions and require `composer dump-autoload`; do not switch to PSR-4 without explicit approval. | SHOULD |
| Unlisted page requested | Validate against `Manager::getPages()` and existing action-page conventions before implementing route changes. | SHOULD |
| Untested behavior changes | Recommend or add targeted PHPUnit coverage in `tests/testsuites/` and note residual risk. | SHOULD |
| Security-sensitive change (auth/session/config) | Prefer minimal changes, preserve local-only assumptions, and document security impact. | MUST |

## 5. Project Stats

- Language / Runtime: PHP `>=8.4`, strict types enabled project-wide.
- Architecture pattern: Page-based PHP app with singleton `Manager`, JSON-file persistence, no MVC framework.
- Package manager: Composer.
- Test framework: PHPUnit (`phpunit/phpunit >= 9.6.9`).
- Build tool: No compile step; dependency/autoload management via Composer (`composer install`, `composer dump-autoload`), static analysis via PHPStan.
- Context docs: CTX Generator (`ctx generate`) — config in `context.yaml` + `assets/classes/**/module-context.yaml`, output in `.context/`.
